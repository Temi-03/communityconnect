# ADR-001: React Native over Flutter

## Date
31-10-2025

## Decision Owner
Tawakalitu Fetuga

---

## Context
Community Connect is a mobile application designed to connect community members by allowing users to create, browse, and accept small local tasks.

The project requires:
- Cross-platform mobile development (iOS and Android)
- Technologies aligned with existing team skills
- Tools that are easy to learn and quick to use due to academic time constraints

Both React Native and Flutter were evaluated as potential frameworks. Architectural differences, development experience, and ecosystem maturity were considered when comparing the two options. Industry explanations and architectural walkthroughs were also reviewed to better understand the trade-offs between these frameworks.

---

## Decision
We decided to use **React Native** instead of **Flutter** for the development of the Community Connect mobile application.

---

## Decision Reasoning
The decision was driven by the following factors:
- Familiarity with JavaScript 
- Faster development and onboarding
- Strong ecosystem and community support
- Ability to share development patterns with future web-based components
- Availability of third-party libraries and integrations

---

## Options Considered

### Option 1: React Native
- Uses JavaScript and React
- Enables cross-platform mobile development using native UI components
- Large and mature community ecosystem

### Option 2: Flutter
- Uses the Dart programming language
- Renders UI using a custom rendering engine
- Strong support and tooling provided by Google

---

## Pros and Cons

### React Native
**Pros**
- Leverages existing JavaScript knowledge
- Faster prototyping and iteration
- Extensive third-party library ecosystem
- Widely adopted in industry

**Cons**
- Bridge-based architecture can introduce performance overhead
- Dependency on native modules for advanced functionality
- Potential maintenance challenges due to library updates

### Flutter
**Pros**
- High performance due to native compilation
- Consistent UI rendering across platforms
- Strong tooling and documentation

**Cons**
- Requires learning Dart
- Smaller ecosystem compared to React Native
- Less alignment with existing project skillset

---

## Consequences
Using React Native allows faster development and easier maintenance within the project timeframe.  
While Flutter may offer performance advantages due to its rendering approach, these benefits were not critical for the scope of this application. The potential performance limitations of React Native are acceptable given the app’s moderate complexity and minimal animation requirements.
